# VoHive 控制台

这是一个独立的 Windows 桌面控制程序，用于管理本机现有的 VoHive WSL 环境。它不会替换 `01_工具入口\VoHive WSL 工具` 中已经可用的脚本或 VoHive 配置；代理管理页只在用户明确保存、导入、删除或切换时更新 WSL 中的 Mihomo 配置。

## 运行要求

- Windows 11，已启用 WSL 2。
- WSL 发行版名称为 `Ubuntu-24.04`。
- 现有工具目录位于桌面 `01_工具入口\VoHive WSL 工具`。
- 已完成 EC20/大疆一代模块、VoHive、Mihomo、QQ Bot 和通话 sidecar 的原有配置。
- 已安装 Microsoft Edge WebView2 Runtime。Windows 11 通常已自带。
- 实时播放模式需要现有环境中已安装 `ffplay.exe`。

## 启动

双击 `VoHive 控制台.exe`，接受一次 Windows 管理员权限提示。管理员权限只用于 USB 挂载、WSL 启动、设备重置和局域网端口配置。

窗口打开后会自动调用现有的 `Start VoHive WSL.ps1 -NoPause`，依次检查并启动：

1. WSL 保持进程和 systemd。
2. EC20 USB 设备挂载状态。
3. Mihomo 前置代理。
4. VoHive 服务和 VoWiFi 注册状态。
5. 固定后台地址 `http://vohive-wsl:7575`。

这些操作均隐藏运行，不显示 PowerShell 或 WSL 终端。界面中的“当前进度”只显示当前步骤，不显示时间戳或原始终端日志。

## 系统托盘后台运行

程序启动后会在 Windows 任务栏右侧通知区域创建 VoHive 图标，图标可能收纳在“显示隐藏的图标”箭头中。
程序图标与当前 VoHive 网页使用的紫色六边形 `V` favicon 保持一致。

- 点击窗口右上角关闭按钮或按 `Alt+F4`：只隐藏主窗口，程序、状态检查和代理自动切换继续在后台运行。
- 双击托盘图标：恢复上次大小的控制台窗口。
- 右键“展开主页面”：恢复窗口并打开控制中心。
- 右键“退出”：完全结束控制台进程；只有这个操作才会退出托盘后台。

Windows 关机或注销时，程序会正常退出，不会阻止系统关闭。

## 控制中心

### 启动 VoHive

手动重新执行一次核心服务检查。服务已经运行时也可以使用，不会另外部署第二份 VoHive。

### 局域网访问

“开启局域网访问”会调用现有的 `Enable VoHive LAN Access.ps1`；再次点击会调用 `Disable VoHive LAN Access.ps1`。状态来自 Windows 当前的 `netsh interface portproxy` 配置，不是仅保存在界面内。

开启后，局域网设备应使用这台 Windows 电脑的局域网 IP 和端口 `7575`，不能使用对方设备自己的 `localhost`。不要把 `7575` 直接暴露到公网。

### 重置 VoHive 设备检测

调用现有的 `Reset VoHive Device Detect.ps1`，重新挂载 EC20 并让 VoHive 重新扫描设备。适用于模块重启后长期停留在“等待设备枚举”的情况。

这个操作只处理设备检测链路，不会修改 SIM、eSIM、Mihomo 节点或 VoHive 业务配置。

## 代理管理

代理管理页直接读取 WSL 的 `/opt/mihomo/config.yaml`。节点数量没有上限，可以逐个添加和编辑，也可以导入以下任一种 Mihomo/Clash YAML：

- 单个节点映射。
- 节点数组。
- 包含 `proxies` 字段的配置片段。

页面只显示名称、协议、服务器、端口及“凭据已保存”状态。已有密码、UUID、Token 和认证字段不会发送给 WebView；编辑节点时凭据留空表示保留原值。

所有节点统一由 Mihomo 输出到 `127.0.0.1:7891`，VoHive 继续把它作为 SOCKS5 前置代理使用。

- **手动切换**：固定使用所选节点，并把自动模式关闭。
- **自动切换**：每 15 秒读取 VoHive 状态；连续 3 次未达到 `tunnel_ready + ims_ready` 后，按列表顺序切到下一个节点。
- **防抖**：每次切换、导入、编辑或删除后保留 90 秒注册时间，避免 Mihomo 重启期间误判。
- **单个测速**：点击节点行内的“测速”，测速前会重新读取 WSL 节点列表；通过节点访问 HTTP 204，最多采集 3 次并显示成功样本的中位延迟。
- **全部测速**：最多同时测试 4 个节点，每个节点分别显示中位延迟或失败状态；测速不会切换当前节点。

测速期间只锁定测速按钮，控制中心和其他代理按钮仍可使用。切换、编辑、删除等需要重启 Mihomo 的操作会等待当前测速结束后自动执行。

首次测速时，控制台会为 Mihomo 配置仅监听 WSL 本机回环地址的控制接口，并生成随机访问密钥。控制接口不会开放到局域网，密钥也不会传给嵌入式网页。页面显示的是代理握手和 HTTP 请求的完整往返延迟（`ms`），不是 ICMP ping 或下载带宽。
默认使用 Google HTTP 204；如果某节点的全部样本都失败，会自动改用 Cloudflare HTTP 204 重试。节点名称通过 UTF-8 传入 WSL，支持国旗 Emoji 和中文。

写入时先生成临时文件并运行 Mihomo 配置校验，通过后才备份和替换正式配置。如果 Mihomo 无法重启，程序会恢复上一份配置。自动监控由托盘常驻的控制台进程执行；只有从托盘菜单选择“退出”后才会停止判断和轮换节点。

## 通话监听

两种模式会先结束旧监听，再启动唯一的一套新监听，因此不会同时抢占 QQ Bot 命令或 RTP 端口。

- **只通话监听**：后台接收通话、保存 WAV 录音并处理 QQ Bot 按键，不在电脑上播放实时声音。
- **播放声音并监听**：在上述功能基础上启动 Windows `ffplay`，通过端口 `19090` 播放实时通话声音。

监听 PowerShell 使用 Explorer 的普通用户主令牌启动，完整性级别为 Medium；控制台本身仍为 High。监听进程使用 `CREATE_NO_WINDOW`，不会出现普通或管理员 PowerShell 窗口。

录音仍由现有监听脚本写入：

`桌面\01_工具入口\VoHive 通话录音\call-年月日-时分秒`

切换模式或重新打开控制台后，界面会从全局监听互斥锁和 `ffplay` 自动识别实际监听状态。关闭主窗口只会隐藏到系统托盘，不会中断监听、VoHive 或 Mihomo。

## VoHive 后台

左侧“VoHive 后台”直接在程序内打开 `http://vohive-wsl:7575`，功能与浏览器访问相同。点击“控制中心”可以返回启动、监听、局域网和设备重置操作。

如果后台暂时打不开，先回到控制中心确认“核心服务”为“服务可用”，再点击“启动服务”。

## 常见状态

- **正在等待 VoHive**：WSL 尚未启动完成，或 `vohive-wsl` 主机名还未更新。
- **通话监听未能切换**：当前进度会显示具体 Windows 错误；旧监听不会被错误替换为管理员监听。
- **实时模式没有声音**：确认 Windows 默认输出设备正常，并检查 `ffplay.exe` 是否存在。
- **局域网凭证失败**：这是 VoHive 后台认证问题，不是端口转发失败；使用 VoHive 当前后台账号登录。

## 项目边界

本目录只包含控制台源码、嵌入式前端和最终 EXE。代理节点、SIM/eSIM 信息、QQ Bot 密钥及 VoHive 私有配置仍保存在原有环境中，不会打包进该程序。
